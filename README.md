# Smart Scrapper Agent

An AI-powered research service that autonomously searches the web, scrapes relevant pages, summarises them with Claude (Anthropic), and returns a structured Markdown report.

## How it works

```
POST /research  →  Plan  →  Act  →  Observe  →  Iterate  →  Report
                   (Claude    (search +    (Claude        (refine
                   generates  scrape +     scores         queries
                   queries)   summarise)   coverage)      if needed)
```

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Set your Anthropic API key (or add it to .env)
echo ANTHROPIC_API_KEY=sk-ant-... > .env

# 3. Start the server
python serve.py

# 4. Submit a research question
curl -X POST http://localhost:8000/research \
     -H "Content-Type: application/json" \
     -d "{\"question\": \"What are the latest breakthroughs in solid-state batteries?\"}"

# 5. Poll for the result (replace JOB_ID with the returned job_id)
curl http://localhost:8000/research/JOB_ID
```

## Running the Server

```bash
# Development (auto-reload on file changes)
python serve.py --reload

# Custom host and port
python serve.py --host 0.0.0.0 --port 9000

# Production (multiple workers)
python serve.py --host 0.0.0.0 --workers 4
```

| Flag | Default | Description |
|------|---------|-------------|
| `--host` | `127.0.0.1` | Bind address (`0.0.0.0` to expose to network) |
| `--port` | `8000` | Bind port |
| `--reload` | off | Auto-reload on code changes (dev only) |
| `--workers` | `1` | Parallel worker processes |

**Interactive docs** (auto-generated by FastAPI):

- Swagger UI: http://localhost:8000/docs
- ReDoc: http://localhost:8000/redoc

### POST /research

Start a new research job. The agent runs in the background; the endpoint returns a `job_id` immediately with HTTP `202 Accepted`.

**Request body**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `question` | `string` | **required** | The research question (min 5 chars) |
| `max_sources` | `integer` | `9` | Max pages to scrape (1–30) |
| `max_iterations` | `integer` | `3` | Max Plan→Act→Observe loops (1–5) |
| `token_budget` | `integer` | `200000` | Max total Claude tokens (min 10000) |
| `model` | `string` | `claude-opus-4-6` | Anthropic model ID |

**Example request**

```json
{
  "question": "What are the latest breakthroughs in solid-state batteries?",
  "max_sources": 5,
  "max_iterations": 2,
  "token_budget": 100000
}
```

**Response `202`**

```json
{
  "job_id": "3f7a2c1e-9b4d-4e8f-a012-1234567890ab",
  "status": "queued",
  "question": "What are the latest breakthroughs in solid-state batteries?",
  "created_at": "2026-02-27T10:30:00+00:00",
  "completed_at": null,
  "report_path": null,
  "report_markdown": null,
  "sources_scraped": null,
  "tokens_used": null,
  "token_budget": 100000,
  "error": null
}
```

### GET /research/{job_id}

Poll a job by its ID. When `status` is `complete`, the full Markdown report is included in `report_markdown`.

**Path parameter:** `job_id` — UUID returned by `POST /research`

**Response `200` — running**

```json
{
  "job_id": "3f7a2c1e-9b4d-4e8f-a012-1234567890ab",
  "status": "running",
  "question": "What are the latest breakthroughs in solid-state batteries?",
  "created_at": "2026-02-27T10:30:00+00:00",
  "completed_at": null,
  "report_markdown": null,
  "sources_scraped": null,
  "tokens_used": null,
  "error": null
}
```

**Response `200` — complete**

```json
{
  "job_id": "3f7a2c1e-9b4d-4e8f-a012-1234567890ab",
  "status": "complete",
  "question": "What are the latest breakthroughs in solid-state batteries?",
  "created_at": "2026-02-27T10:30:00+00:00",
  "completed_at": "2026-02-27T10:32:45+00:00",
  "report_path": "C:/...output/report_20260227_103245.md",
  "report_markdown": "# Research Report\n\n**Question:** ...",
  "sources_scraped": 5,
  "tokens_used": 42381,
  "token_budget": 100000,
  "error": null
}
```

**Response `200` — failed**

```json
{
  "status": "failed",
  "error": "Network error: connection refused"
}
```

**Response `404`** — job ID not found.

### GET /research/{job_id}/report

Download the Markdown report as a `.md` file. Only available when `status` is `complete`.

**Response `200`** — `Content-Type: text/markdown` file download.

**Response `409`** — job not yet complete.

**Response `404`** — job not found or report file missing.

---

### DELETE /research/{job_id}

Remove a job from the in-memory store. Cannot delete a running job.

**Response `204`** — deleted.

**Response `409`** — job is still running.

**Response `404`** — job not found.

---

### POST /scrape

Scrape a single URL synchronously. Retries on transient errors with exponential backoff.

**Request body**

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `url` | `string` | **required** | Full URL to fetch |
| `timeout` | `integer` | `20` | Per-attempt timeout in seconds (5–60) |
| `max_retries` | `integer` | `3` | Max retry attempts (1–5) |

**Example request**

```json
{
  "url": "https://en.wikipedia.org/wiki/Solid-state_battery",
  "timeout": 15,
  "max_retries": 3
}
```

**Response `200`**

```json
{
  "url": "https://en.wikipedia.org/wiki/Solid-state_battery",
  "title": "Solid-state battery - Wikipedia",
  "text": "A solid-state battery is a battery technology...",
  "char_count": 18432,
  "success": true,
  "error": "",
  "attempts": 1
}
```

### Python (requests)

```python
import time
import requests

BASE = "http://localhost:8000"

# ── Start a research job ───────────────────────────────────────────────────────
response = requests.post(f"{BASE}/research", json={
    "question": "What are the latest breakthroughs in solid-state batteries?",
    "max_sources": 5,
    "max_iterations": 2,
    "token_budget": 100_000,
})
response.raise_for_status()
job = response.json()
job_id = job["job_id"]
print(f"Job started: {job_id}")

# ── Poll until complete ────────────────────────────────────────────────────────
while True:
    result = requests.get(f"{BASE}/research/{job_id}").json()
    status = result["status"]
    print(f"Status: {status}")

    if status == "complete":
        print("\n── Report ──────────────────────────────")
        print(result["report_markdown"])
        print(f"\nSources scraped : {result['sources_scraped']}")
        print(f"Tokens used     : {result['tokens_used']:,} / {result['token_budget']:,}")
        break

    if status == "failed":
        print(f"Job failed: {result['error']}")
        break

    time.sleep(5)  # poll every 5 seconds

# ── Download report file ───────────────────────────────────────────────────────
if result["status"] == "complete":
    report = requests.get(f"{BASE}/research/{job_id}/report")
    with open("my_report.md", "wb") as f:
        f.write(report.content)
    print("Report saved to my_report.md")

# ── Scrape a single URL ────────────────────────────────────────────────────────
scrape = requests.post(f"{BASE}/scrape", json={
    "url": "https://en.wikipedia.org/wiki/Solid-state_battery",
    "timeout": 15,
}).json()

if scrape["success"]:
    print(f"Scraped {scrape['char_count']:,} chars from: {scrape['title']}")
```

### JavaScript (fetch)

```javascript
const BASE = "http://localhost:8000";

// Start a research job
const res = await fetch(`${BASE}/research`, {
  method: "POST",
  headers: { "Content-Type": "application/json" },
  body: JSON.stringify({
    question: "What are the latest breakthroughs in solid-state batteries?",
    max_sources: 5,
    max_iterations: 2,
  }),
});
const { job_id } = await res.json();
console.log("Job started:", job_id);

// Poll until complete
const poll = async () => {
  const result = await fetch(`${BASE}/research/${job_id}`).then(r => r.json());

  if (result.status === "complete") {
    console.log("Report:\n", result.report_markdown);
    console.log(`Sources: ${result.sources_scraped}, Tokens: ${result.tokens_used}`);
    return;
  }
  if (result.status === "failed") {
    console.error("Failed:", result.error);
    return;
  }

  console.log("Status:", result.status, "— retrying in 5s");
  setTimeout(poll, 5000);
};

poll();
```

## Job Lifecycle

```
POST /research
      │
      ▼
   [queued]  ──► background thread starts
      │
      ▼
   [running] ──► PLAN → ACT → OBSERVE → ITERATE → REPORT
      │                                              │
      ▼                                              ▼
  [complete]                                     [failed]
  report_markdown populated               error message populated
```

Typical duration: **30 seconds – 3 minutes** depending on `max_sources`, `max_iterations`, and network latency.

## Error Reference

| HTTP Status | Meaning |
|-------------|---------|
| `202` | Job accepted and queued |
| `400` | Invalid request body or query parameter |
| `404` | Job ID not found, or report file missing |
| `409` | Conflict — e.g. deleting a running job, or downloading a report not yet ready |
| `422` | Validation error — check request body field types and constraints |
| `500` | Unexpected server error |

## Configuration

All agent behaviour is controlled per-request via the `POST /research` body. There are no global configuration files — each job is fully self-contained.

| Parameter | Min | Max | Default | Effect |
|-----------|-----|-----|---------|--------|
| `max_sources` | 1 | 30 | 9 | More sources → more thorough but slower and costlier |
| `max_iterations` | 1 | 5 | 3 | More iterations → agent refines gaps in coverage |
| `token_budget` | 10,000 | — | 200,000 | Hard cap on Anthropic API spend per job |
| `model` | — | — | `claude-opus-4-6` | Swap to `claude-haiku-4-5-20251001` for faster/cheaper runs |

**Recommended settings by use case:**

| Use case | `max_sources` | `max_iterations` | `model` |
|----------|---------------|-----------------|---------|
| Quick overview | 3 | 1 | `claude-haiku-4-5-20251001` |
| Standard research | 9 | 3 | `claude-opus-4-6` |
| Deep research | 20 | 5 | `claude-opus-4-6` |
